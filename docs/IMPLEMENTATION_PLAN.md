# English Coach — план реализации

> Форк [m98/fluent](https://github.com/m98/fluent) → [alklimover/english-coach](https://github.com/alklimover/english-coach).
> Личный voice-first тренажёр английского в терминале (Claude Code), с недельной программой занятий.
> Дата плана: 2026-07-07. Базовый коммит форка: `86fb80f` (v0.3.0).

---

## 1. Цель и принципы

**Цель:** заговорить на английском. Инструмент — средство, не цель.

**Метрика успеха инструмента:** 20 голосовых сессий за первый месяц использования (≥3 в неделю). Не «сколько фич сделано».

**Принципы:**

1. **Voice-first.** Говорение — главная активность, всё остальное её обслуживает.
2. **Коуч, а не чат.** Продукт ведёт по недельному плану (speaking / listening / writing / review), а не ждёт запроса.
3. **Не ломать upstream.** Новые скиллы добавляем своими директориями, существующие файлы fluent меняем минимально — чтобы `git fetch upstream` оставался дешёвым.
4. **Данные переносимы.** Память ученика — JSON-файлы, команды — markdown-промпты. Сегодня runtime — Claude Code, завтра может быть любой агент или отдельное приложение. Ничего не привязываем к конкретной модели.
5. **Инженерка не съедает практику.** После Фазы 2 — первая реальная сессия. Фаза 3 начинается только после 5 проведённых голосовых сессий.

## 2. Архитектура

### 2.1 Слои

```
┌─────────────────────────────────────────────────┐
│  Coach layer (новое)                            │
│  /coach-plan  /coach-today  /discuss            │
│  data/weekly-plan.json                          │
├─────────────────────────────────────────────────┤
│  Voice layer (новое)                            │
│  /talk — живой разговор                         │
│  вход: Handy (диктовка) · выход: TTS (say→Kokoro)│
├─────────────────────────────────────────────────┤
│  Fluent core (upstream, почти не трогаем)       │
│  6 JSON-баз · SM-2 · хуки read-db/update-db     │
│  /fluent-review /fluent-writing /fluent-vocab … │
└─────────────────────────────────────────────────┘
```

### 2.2 Голосовой контур

- **Вход (STT):** [Handy](https://github.com/cjpais/Handy) — уже установлен. Хоткей → говоришь по-английски → текст падает в промпт Claude Code. Ничего не кодим.
- **Выход (TTS):** MVP — macOS `say` с Siri-голосом (офлайн, мгновенно). Апгрейд в Фазе 4 — Kokoro через `mlx-audio` (естественнее). Вызов TTS — Bash-команда внутри скилла после каждой реплики Claude.
- **Следствие для скиллов:** реплики Claude в голосовых сессиях — короткие (2–3 предложения), разговорные, **без markdown** (озвучиваются как есть).

### 2.3 Данные

Все базы fluent остаются как есть (`learner-profile`, `progress-db`, `mistakes-db`, `mastery-db`, `spaced-repetition`, `session-log` + `results/*.md`). Добавляется одна новая: `weekly-plan.json` (схема — §5.1).

**Приватность:** форк публичный (GitHub не позволяет приватить форки). Личные данные (профиль, ошибки, логи сессий) **не коммитим**:

- `data/` и `results/` — в `.gitignore` (проверить, дополнить);
- бэкап данных — локальный механизм fluent (`session-end.py` делает снапшоты) + позже приватный репо `english-coach-data` или iCloud (созвучно KR3.2 про зеркала).

## 3. Фаза 0 — гигиена форка и багфиксы

*Оценка: ~30 минут.*

### 0.1 Настроить remote upstream

```bash
git remote add upstream https://github.com/m98/fluent
git fetch upstream
```

**Критерий:** `git log upstream/main` работает; понятно, как черри-пикать обновления.

### 0.2 Багфикс `update-db.py` (из открытого upstream PR #11)

Проблема: `main()` читает JSON **только из stdin** (`.claude/hooks/update-db.py:534`). При вызове с payload в argv базы молча не обновляются. Фикс — 3 строки (как в PR #11, но сам PR тащит ещё Anki-экспорт и import-lr, их не берём):

```python
if len(sys.argv) > 1:
    session = json.loads(sys.argv[1])
else:
    session = json.load(sys.stdin)
```

Дополнительно: тест-кейс на argv-путь в `tests/test_update_db.py`.

**Критерий:** `pytest tests/` зелёный; вызов `update-db.py '<json>'` и `echo '<json>' | update-db.py` дают одинаковый результат.

### 0.3 Идентичность форка

- `.claude-plugin/plugin.json`: `name: english-coach`, версия `0.3.0-ec.1`, homepage/repository → наш репо.
- `README.md`: шапка — что это форк fluent, заточенный под английский + голос + недельный план; ссылка на upstream.
- Режим установки: работаем как **папка-проект** (сессия запускается из `english-coach/`), не как plugin install. Хуки в `.claude/settings.json` уже настроены на это.

**Критерий:** новая сессия Claude Code из папки видит хуки (SessionStart печатает статистику) и скиллы.

### 0.4 Приватность данных

- Проверить `.gitignore`: `data/*.json`, `results/*.md`, бэкапы — игнорируются (`.gitkeep` остаются).

**Критерий:** `git status` после тестовой сессии не показывает личных файлов.

## 4. Фаза 1 — персонализация под английский

*Оценка: ~1 час.*

### 1.1 Профиль

Прогнать `/fluent-setup`: `target_language: English`, `native_language: Russian`, текущий уровень (определить в первой сессии разговором, не тестом), целевой уровень, `daily_goal_minutes`, мотивация: work (созвоны, питчи, small talk, формулировать мысль без русского костыля).

В `preferences` добавить свои поля (fluent их не тронет):

```json
"voice": {
  "tts_enabled": true,
  "tts_voice": "…выбранный Siri-голос…",
  "tts_rate": 180,
  "input": "handy"
}
```

### 1.2 Сид типичных паттернов русскоязычных

В `mistakes-db.json` завести паттерны-«вахтёры» с `frequency: 0` и пометкой в `notes: "typical RU-speaker pattern — watch"`:
артикли (a/the/∅), Present Perfect vs Past Simple, предлоги (depends **on**, listen **to**), порядок слов в вопросах, will после if, кальки («I feel myself», «very actual»), countable/uncountable (advices, informations).

**Важно:** frequency не выдумываем — счётчики растут только от реальных ошибок в сессиях.

### 1.3 Английские примеры в скиллах

В `fluent-speaking`, `fluent-learn`, `fluent-vocab`: голландские примеры и conversational fillers заменить на английские (well…, actually…, to be fair…, that makes sense). Изменения — точечные, в секциях примеров (см. принцип №3).

### 1.4 Локальный контекст в `CLAUDE.md` форка

Добавить секцию (аддитивно, в конец): ученик один ({LEARNER_NAME}), язык EN, вход надиктован через Handy, выход озвучивается TTS, язык объяснений — английский по умолчанию, русский по запросу или для критически непонятых мест.

**Критерий фазы:** `/fluent-review` и `/fluent-vocab` работают на английском контенте, профиль заполнен, сид-паттерны в базе.

## 5. Фаза 2 — голосовой слой: `/talk`

*Оценка: 2–3 часа + первая тест-сессия. Самая важная фаза.*

### 2.1 Скилл `.claude/skills/talk/SKILL.md`

Frontmatter: `disable-model-invocation: true`, `allowed-tools: Read, Write, Bash`.

**Поток сессии:**

1. **Контекст:** `read-db.py` → уровень, слабые места, due-фразы из SRS (вплести 2–3 в разговор).
2. **Сценарий:** из `weekly-plan.json` (если есть план на сегодня), иначе — предложить 3 на выбор по слабым темам. Библиотека сценариев: созвон с инвестором, статус-митинг, small talk, объяснить продукт, спор/отстаивание позиции, собеседование, travel/бытовые.
3. **Разговор (ядро):**
   - реплики Claude: 2–3 предложения, разговорный тон, без markdown, без эмодзи;
   - после каждой реплики — озвучка: `say -v "$TTS_VOICE" -r $TTS_RATE "…"`;
   - **никаких оценок в потоке** — ошибки и удачные фразы копятся молча;
   - вход надиктован: игнорировать пунктуацию/капитализацию; явно бессвязный фрагмент → переспросить как собеседник («Sorry, you mean…?»), в ошибки не писать;
   - если ученик застрял — подсказать фразу, продолжить;
   - длительность ~10–15 минут или до стоп-фразы («let's wrap up»).
4. **Разбор (после разговора):**
   - топ-3 ошибки с объяснениями → `mistakes-db` (только реальные, communication-blocking или повторяющиеся);
   - 5 полезных/натуральных фраз → `spaced-repetition` (type: phrase);
   - 1 фокус на следующую сессию;
   - оценка по схеме fluent (communication-first) → mastery speaking.
5. **Персист:** `fluent-db-updater` (`command_used: "/talk"`, `skills_practiced: ["speaking"]`), транскрипт → `results/talk-session-{NNN}.md`.

### 2.2 TTS-настройка

- `say -v '?'` — посмотреть установленные голоса; скачать премиум/Siri-голос в System Settings → Accessibility → Spoken Content.
- Голос и темп — из `learner-profile.preferences.voice` (§1.1).
- Fallback: `tts_enabled: false` → сессия текстом (не блокирует практику, если TTS бесит).

### 2.3 Первая тест-сессия и калибровка

Провести полный `/talk` (≥10 минут). Проверить и подкрутить:

- качество распознавания Handy на английском с акцентом (если плохо → в настройках Handy модель покрупнее, например Whisper large; крайний fallback — встроенный `/voice` Claude Code);
- длина реплик и темп `say` (не утомляет ли);
- латентность хода (цель: ≤5 секунд от конца твоей реплики до начала озвучки ответа);
- качество разбора и записей в базах (`read-db.py` после сессии).

**Критерий фазы:** полный голосовой разговор ≥10 минут → разбор → базы обновлены → транскрипт в `results/`. Это и есть **v0 готов**.

## 6. Фаза 3 — coach-слой: недельная программа

*Оценка: 3–4 часа. Старт — после 5 проведённых `/talk`-сессий (принцип №5).*

### 3.1 Схема `data/weekly-plan.json`

```json
{
  "week_id": "2026-W29",
  "created": "2026-07-13",
  "goals": ["уверенно пройти сценарий 'investor call'", "закрыть паттерн articles в топ-3"],
  "activities": [
    {
      "id": "w29-1",
      "day": "mon",
      "type": "talk",
      "scenario": "status update call",
      "status": "planned",
      "session_ref": null
    },
    {
      "id": "w29-2",
      "day": "tue",
      "type": "listen",
      "source": {"title": "…", "url": "…", "duration_min": 20},
      "followup": "w29-3",
      "status": "planned"
    },
    {
      "id": "w29-3",
      "day": "wed",
      "type": "discuss",
      "about": "w29-2",
      "status": "planned"
    },
    {"id": "w29-4", "day": "thu", "type": "write", "topic": "…", "status": "planned"},
    {"id": "w29-5", "day": "fri", "type": "review", "status": "planned"},
    {"id": "w29-6", "day": "sat", "type": "talk", "scenario": "small talk", "status": "planned"}
  ],
  "retro": {"completed_pct": null, "carried_over": [], "notes": ""}
}
```

`type ∈ {talk, listen, discuss, write, review, vocab}`; `status ∈ {planned, done, skipped}`. Каждый скилл-активность по завершении отмечает свою запись `done` и ставит `session_ref`.

### 3.2 Скилл `/coach-plan`

1. Читает базы: слабые навыки (mastery), топ паттернов ошибок (mistakes), объём SRS-очереди, `retro` прошлой недели (что пропущено → перенести/облегчить).
2. Короткое интервью: занятость недели, чего хочется/что бесило.
3. Генерит план: 3–4 talk, 1 listen+discuss (подбирает материал по интересам и уровню: подкаст/видео/выступление), 1 write, 1–2 review. Общая нагрузка ≤ `daily_goal_minutes` × доступные дни.
4. Показывает таблицей, правки — диалогом, пишет `weekly-plan.json`.

Каданс — как в Chaush OS: воскресенье ретро → понедельник план.

### 3.3 Скилл `/coach-today`

Читает план + сегодняшний день → «сегодня по плану X, погнали?» → запускает соответствующий скилл. Если день пропущен вчера — мягко предлагает нагнать или перепланировать (без вины: правило «пропуск — это данные, не провал»).

### 3.4 SessionStart-хук: план дня в приветствии

`session-start.py` дополнить (аддитивно, в конец вывода): если есть `weekly-plan.json` и активность на сегодня — вывести одной строкой. Терминал открылся → сразу видно, что делать.

### 3.5 Скилл `/discuss`

Обсуждение listening-материала из плана:

- **до** просмотра (в момент планирования): 3 вопроса-ориентира, на что обратить внимание;
- **после**: голосовое обсуждение по механике `/talk` (TTS + Handy) — пересказ, мнение, спор с тезисами автора;
- новая лексика из материала → SRS; разбор — как в `/talk`.

### 3.6 Writing — используем существующий

`/fluent-writing` не переписываем; `/coach-today` передаёт ему тему из плана. Темы — рабочие: письмо инвестору, апдейт команде, ответ на возражение.

**Критерий фазы:** полный цикл недели: `/coach-plan` → активности выполняются и помечаются `done` → следующий `/coach-plan` видит retro и адаптирует программу.

## 8. Риски

| Риск | Митигация |
|------|-----------|
| Залипнуть в инженерку, не говорить | Фаза 2 — сразу после Фазы 0–1; Фаза 3 — только после 5 живых сессий; метрика — сессии, не фичи |
| `say` звучит роботом, бесит | приемлемо для MVP; Kokoro в 4.1; `tts_enabled: false` как аварийный выход |
| Handy плохо распознаёт акцент | крупнее модель в Handy; бенчмарк 4.2; fallback `/voice` |
| Оценки ломают разговорный поток | жёсткое правило в `/talk`: ноль фидбека до конца разговора |
| Личные данные в публичном форке | `data/`, `results/` в `.gitignore` (Фаза 0.4) |
| Дивергенция с upstream | новое — новыми файлами; правки существующих — точечные; remote upstream с Фазы 0.1 |
| План на неделю давит, привычка умирает | нагрузка ≤ daily_goal; пропуск = данные; `/coach-plan` умеет облегчать неделю |
